# Building G'MIC-Qt Plugin for GIMP 3.2.4 on macOS Sequoia (arm64)

This guide documents the complete process to build and install the G'MIC-Qt plugin
for GIMP 3.2.4 installed via Homebrew Cask on macOS Sequoia (Apple Silicon / arm64).

---

## Prerequisites

- macOS Sequoia (arm64 / Apple Silicon)
- GIMP 3.2.4 installed via `brew install --cask gimp`
- Homebrew installed at `/opt/homebrew`
- Xcode Command Line Tools installed

---

## Step 1 â€” Install build dependencies via Homebrew

```bash
brew install qt pkg-config meson ninja \
  babl gegl glib gtk4 cairo pango gdk-pixbuf \
  harfbuzz fontconfig freetype2 gexiv2 \
  libpng jpeg-turbo fftw
```

---

## Step 2 â€” Clone gmic and gmic-qt sources

```bash
mkdir -p ~/PhotoApps
cd ~/PhotoApps
git clone https://github.com/GreycLab/gmic.git
git clone https://github.com/c-koi/gmic-qt.git
```

The gmic-qt source must be a sibling of the gmic source directory
(gmic-qt expects to find gmic sources at `../gmic/src`).

---

## Step 3 â€” Download GIMP 3.2.4 source tarball (for headers only)

The GIMP Homebrew Cask installs an app bundle without development headers.
We need the source tarball solely for its header files.

```bash
curl -fLO https://download.gimp.org/gimp/v3.2/gimp-3.2.4.tar.xz
tar xf gimp-3.2.4.tar.xz -C ~
```

This extracts to `~/gimp-3.2.4`.

> **Important:** Use `curl -fLO`, not `curl -O`. `download.gimp.org` returns a
> 302 redirect to a mirror; without `-L` curl follows nothing and writes a
> **0-byte** file. `tar xf` on an empty file silently succeeds and extracts
> nothing, so no headers land on disk — and the build fails much later with
> `'libgimp/gimp.h' file not found`. The `-f` flag makes curl fail loudly on
> HTTP errors. After extracting, verify with: `ls ~/gimp-3.2.4/libgimp/gimp.h`

## Step 4 â€” Manually generate gimpversion.h

This file is normally generated by meson during a GIMP build.
Since we are not building GIMP, create it manually:

```bash
mkdir -p ~/gimp-3.2.4/_build/libgimpbase

cat > ~/gimp-3.2.4/_build/libgimpbase/gimpversion.h << 'EOF'
#ifndef __GIMP_VERSION_H__
#define __GIMP_VERSION_H__

#if !defined (__GIMP_BASE_H_INSIDE__) && !defined (GIMP_BASE_COMPILATION)
#error "Only <libgimpbase/gimpbase.h> can be included directly."
#endif

G_BEGIN_DECLS

#define GIMP_MAJOR_VERSION                              (3)
#define GIMP_MINOR_VERSION                              (2)
#define GIMP_MICRO_VERSION                              (4)
#define GIMP_VERSION                                    "3.2.4"
#define GIMP_API_VERSION                                "3.0"

#define GIMP_CHECK_VERSION(major, minor, micro) \
    (GIMP_MAJOR_VERSION > (major) || \
     (GIMP_MAJOR_VERSION == (major) && GIMP_MINOR_VERSION > (minor)) || \
     (GIMP_MAJOR_VERSION == (major) && GIMP_MINOR_VERSION == (minor) && \
      GIMP_MICRO_VERSION >= (micro)))

G_END_DECLS

#endif /* __GIMP_VERSION_H__ */
EOF
```

## Step 5 â€” Create a gimp-3.0.pc pkg-config file

The GIMP bundle does not include `.pc` files. Create one manually.
Note: run these commands as your regular user (not root/sudo).

```bash
mkdir -p ~/lib/pkgconfig

GIMP_BUNDLE=/Applications/GIMP.app/Contents/Resources
GIMP_SRC=~/gimp-3.2.4
BREW=/opt/homebrew

cat > ~/lib/pkgconfig/gimp-3.0.pc << EOF
prefix=${GIMP_BUNDLE}
libdir=\${prefix}/lib
includedir=${GIMP_SRC}

Name: GIMP
Description: GNU Image Manipulation Program
Version: 3.2.4
Libs: -L\${libdir} -lgimp-3.0 -lgimpbase-3.0 -lgimpcolor-3.0 -lgimpconfig-3.0 -lgimpmath-3.0 -lgimpmodule-3.0 -lgimpthumb-3.0 -lgimpwidgets-3.0 -lgimpui-3.0
Cflags: -I\${includedir} -I\${includedir}/_build \
  -I${BREW}/opt/glib/include/glib-2.0 \
  -I${BREW}/opt/glib/lib/glib-2.0/include \
  -I${BREW}/opt/glib/include/gio-unix-2.0 \
  -I${BREW}/opt/cairo/include/cairo \
  -I${BREW}/opt/pango/include/pango-1.0 \
  -I${BREW}/opt/gdk-pixbuf/include/gdk-pixbuf-2.0 \
  -I${BREW}/opt/gtk4/include/gtk-4.0 \
  -I${BREW}/opt/harfbuzz/include/harfbuzz \
  -I${BREW}/opt/fontconfig/include \
  -I${BREW}/opt/freetype/include/freetype2 \
  -I${BREW}/opt/gegl/include/gegl-0.4 \
  -I${BREW}/opt/babl/include/babl-0.1 \
  -I${BREW}/opt/gexiv2/include/gexiv2-0.16
EOF
```

Add to your `~/.zshrc` to make permanent:

```bash
echo 'export PKG_CONFIG_PATH="$HOME/lib/pkgconfig:/opt/homebrew/lib/pkgconfig"' >> ~/.zshrc
export PKG_CONFIG_PATH="$HOME/lib/pkgconfig:/opt/homebrew/lib/pkgconfig"
```

Verify:

```bash
pkg-config --modversion gimp-3.0   # should print 3.2.4
```

## Step 6 â€” Patch host_gimp.cpp for GIMP 3.2 API changes

GIMP 3.2 introduced breaking API changes not yet reflected in gmic-qt v3.4.2.
Apply the following patch script:

```bash
cd ~/PhotoApps/gmic-qt

cat > /tmp/patch_gmic.py << 'EOF'
with open('src/Host/Gimp/host_gimp.cpp', 'r') as f:
    content = f.read()

# Fix 1: forward declaration - add GIMP 3.2 branch (no n_drawables)
content = content.replace(
    '#if !GIMP_CHECK_VERSION(2, 99, 19)\nstatic GimpValueArray * gmic_qt_run(GimpProcedure * procedure, GimpRunMode run_mode, GimpImage * image, gint n_drawables, GimpDrawable ** drawables, const GimpValueArray * args, gpointer run_data);\n#else\nstatic GimpValueArray * gmic_qt_run(GimpProcedure * procedure, GimpRunMode run_mode, GimpImage * image, gint n_drawables, GimpDrawable ** drawables, GimpProcedureConfig *config, gpointer run_data);\n#endif',
    '#if !GIMP_CHECK_VERSION(2, 99, 19)\nstatic GimpValueArray * gmic_qt_run(GimpProcedure * procedure, GimpRunMode run_mode, GimpImage * image, gint n_drawables, GimpDrawable ** drawables, const GimpValueArray * args, gpointer run_data);\n#elif !GIMP_CHECK_VERSION(3, 2, 0)\nstatic GimpValueArray * gmic_qt_run(GimpProcedure * procedure, GimpRunMode run_mode, GimpImage * image, gint n_drawables, GimpDrawable ** drawables, GimpProcedureConfig *config, gpointer run_data);\n#else\nstatic GimpValueArray * gmic_qt_run(GimpProcedure * procedure, GimpRunMode run_mode, GimpImage * image, GimpDrawable ** drawables, GimpProcedureConfig *config, gpointer run_data);\n#endif'
)

# Fix 2: actual definition - add GIMP 3.2 branch
content = content.replace(
    '#if !GIMP_CHECK_VERSION(2, 99, 19)\nstatic GimpValueArray * gmic_qt_run(GimpProcedure * procedure, GimpRunMode run_mode, GimpImage * image, gint n_drawables, GimpDrawable ** drawables, const GimpValueArray * args, gpointer run_data)\n#else\nstatic GimpValueArray * gmic_qt_run(GimpProcedure * procedure, GimpRunMode run_mode, GimpImage * image, gint n_drawables, GimpDrawable ** drawables, GimpProcedureConfig *config, gpointer run_data)\n#endif',
    '#if !GIMP_CHECK_VERSION(2, 99, 19)\nstatic GimpValueArray * gmic_qt_run(GimpProcedure * procedure, GimpRunMode run_mode, GimpImage * image, gint n_drawables, GimpDrawable ** drawables, const GimpValueArray * args, gpointer run_data)\n#elif !GIMP_CHECK_VERSION(3, 2, 0)\nstatic GimpValueArray * gmic_qt_run(GimpProcedure * procedure, GimpRunMode run_mode, GimpImage * image, gint n_drawables, GimpDrawable ** drawables, GimpProcedureConfig *config, gpointer run_data)\n#else\nstatic GimpValueArray * gmic_qt_run(GimpProcedure * procedure, GimpRunMode run_mode, GimpImage * image, GimpDrawable ** drawables, GimpProcedureConfig *config, gpointer run_data)\n#endif'
)

# Fix 3: gimp_image_get_layers - new API returns GimpLayer** with no count arg
content = content.replace(
    '  _GimpLayerPtr * layers = gimp_image_get_layers(imageId, &layersCount);',
    '#if GIMP_CHECK_VERSION(3, 2, 0)\n  _GimpLayerPtr * layers = gimp_image_get_layers(imageId);\n  if (layers) { while (layers[layersCount]) layersCount++; }\n#else\n  _GimpLayerPtr * layers = gimp_image_get_layers(imageId, &layersCount);\n#endif'
)

# Fix 4: gimp_item_get_children - same pattern
content = content.replace(
    '      _GimpItemPtr * children = gimp_item_get_children(_GIMP_ITEM(idStack.top()), &childCount);',
    '#if GIMP_CHECK_VERSION(3, 2, 0)\n      _GimpItemPtr * children = gimp_item_get_children(_GIMP_ITEM(idStack.top()));\n      if (children) { while (children[childCount]) childCount++; }\n#else\n      _GimpItemPtr * children = gimp_item_get_children(_GIMP_ITEM(idStack.top()), &childCount);\n#endif'
)

# Fix 5: gimp_image_set_selected_layers - remove count argument
content = content.replace(
    '        gimp_image_set_selected_layers(gmic_qt_gimp_image_id, 1, (const GimpLayer **)&selected_layer);',
    '#if GIMP_CHECK_VERSION(3, 2, 0)\n        gimp_image_set_selected_layers(gmic_qt_gimp_image_id, (const GimpLayer **)&selected_layer);\n#else\n        gimp_image_set_selected_layers(gmic_qt_gimp_image_id, 1, (const GimpLayer **)&selected_layer);\n#endif'
)

# Fix 6: gimp_image_procedure_new - explicit cast for GIMP 3.2
content = content.replace(
    '    procedure = gimp_image_procedure_new(plug_in, name, GIMP_PDB_PROC_TYPE_PLUGIN, gmic_qt_run, NULL, NULL);',
    '#if GIMP_CHECK_VERSION(3, 2, 0)\n    procedure = gimp_image_procedure_new(plug_in, name, GIMP_PDB_PROC_TYPE_PLUGIN,\n      (GimpRunImageFunc)gmic_qt_run, NULL, NULL);\n#else\n    procedure = gimp_image_procedure_new(plug_in, name, GIMP_PDB_PROC_TYPE_PLUGIN, gmic_qt_run, NULL, NULL);\n#endif'
)

with open('src/Host/Gimp/host_gimp.cpp', 'w') as f:
    f.write(content)

print("Patch applied successfully.")
EOF

python3 /tmp/patch_gmic.py
```

## Step 7 â€” Build gmic-qt

```bash
cd ~/PhotoApps/gmic-qt

export GIMP_SRC=~/gimp-3.2.4
export GIMP_BUNDLE=/Applications/GIMP.app/Contents/Resources
export BREW=/opt/homebrew
export PKG_CONFIG_PATH="$HOME/lib/pkgconfig:$BREW/lib/pkgconfig"

qmake HOST=gimp3 \
  QMAKE_CXXFLAGS+="-I${GIMP_SRC} -I${GIMP_SRC}/_build \
    -I${BREW}/opt/glib/include/glib-2.0 \
    -I${BREW}/opt/glib/lib/glib-2.0/include \
    -I${BREW}/opt/glib/include/gio-unix-2.0 \
    -I${BREW}/opt/cairo/include/cairo \
    -I${BREW}/opt/pango/include/pango-1.0 \
    -I${BREW}/opt/gdk-pixbuf/include/gdk-pixbuf-2.0 \
    -I${BREW}/opt/gtk4/include/gtk-4.0 \
    -I${BREW}/opt/harfbuzz/include/harfbuzz \
    -I${BREW}/opt/fontconfig/include \
    -I${BREW}/opt/freetype/include/freetype2 \
    -I${BREW}/opt/gegl/include/gegl-0.4 \
    -I${BREW}/opt/babl/include/babl-0.1 \
    -I${BREW}/opt/gexiv2/include/gexiv2-0.16" \
  QMAKE_LFLAGS+="-L${BREW}/opt/glib/lib \
    -L${BREW}/opt/gegl/lib \
    -L${BREW}/opt/babl/lib \
    -L${GIMP_BUNDLE}/lib \
    -Wl,-rpath,/Applications/GIMP.app/Contents/Resources \
    -Wl,-rpath,/opt/homebrew/opt/libpng/lib \
    -Wl,-rpath,/opt/homebrew/opt/jpeg-turbo/lib \
    -Wl,-rpath,/opt/homebrew/opt/fftw/lib \
    -Wl,-alias,_g_once_init_enter,_g_once_init_enter_pointer \
    -Wl,-alias,_g_once_init_leave,_g_once_init_leave_pointer" \
  LIBS="-lgimp-3.0 -lgimpbase-3.0 -lgimpcolor-3.0 -lgimpconfig-3.0 \
    -lgimpmath-3.0 -lgimpmodule-3.0 -lgimpthumb-3.0 \
    -lgimpwidgets-3.0 -lgimpui-3.0 \
    -lbabl-0.1 -lgegl-0.4 -lglib-2.0 -lgobject-2.0 -lgio-2.0"

make
```

> **Note on the `-Wl,-alias` flags:** The GIMP bundle's GLib was built with an
> older version that lacks `g_once_init_enter_pointer` and
> `g_once_init_leave_pointer`. These linker aliases redirect those symbols to
> the older `g_once_init_enter` / `g_once_init_leave` that the bundle's GLib
> does export, producing a clean binary without any post-build patching.

## Step 8 â€” Fix library paths in the built binary

The built binary references some libraries via wrong paths. Fix them with
`install_name_tool`:

```bash
PLUGIN=~/PhotoApps/gmic-qt/gmic_gimp_qt.app/Contents/MacOS/gmic_gimp_qt
BUNDLE=/Applications/GIMP.app/Contents/Resources
BREW=/opt/homebrew

# Redirect GLib/GEGL/babl from Homebrew to GIMP bundle
install_name_tool -change /opt/homebrew/opt/glib/lib/libglib-2.0.0.dylib \
  ${BUNDLE}/lib/libglib-2.0.0.dylib "$PLUGIN"
install_name_tool -change /opt/homebrew/opt/glib/lib/libgobject-2.0.0.dylib \
  ${BUNDLE}/lib/libgobject-2.0.0.dylib "$PLUGIN"
install_name_tool -change /opt/homebrew/opt/glib/lib/libgio-2.0.0.dylib \
  ${BUNDLE}/lib/libgio-2.0.0.dylib "$PLUGIN"
install_name_tool -change /opt/homebrew/opt/babl/lib/libbabl-0.1.0.dylib \
  ${BUNDLE}/lib/libbabl-0.1.0.dylib "$PLUGIN"
install_name_tool -change /opt/homebrew/opt/gegl/lib/libgegl-0.4.0.dylib \
  ${BUNDLE}/lib/libgegl-0.4.0.dylib "$PLUGIN"

# Fix @rpath/lib/ entries for GIMP libraries
install_name_tool -change @rpath/lib/libcurl.4.dylib \
  ${BUNDLE}/lib/libcurl.4.dylib "$PLUGIN"
install_name_tool -change @rpath/lib/libgimp-3.0.0.dylib \
  ${BUNDLE}/lib/libgimp-3.0.0.dylib "$PLUGIN"
install_name_tool -change @rpath/lib/libgimpbase-3.0.0.dylib \
  ${BUNDLE}/lib/libgimpbase-3.0.0.dylib "$PLUGIN"
install_name_tool -change @rpath/lib/libgimpcolor-3.0.0.dylib \
  ${BUNDLE}/lib/libgimpcolor-3.0.0.dylib "$PLUGIN"
install_name_tool -change @rpath/lib/libgimpconfig-3.0.0.dylib \
  ${BUNDLE}/lib/libgimpconfig-3.0.0.dylib "$PLUGIN"
install_name_tool -change @rpath/lib/libgimpmath-3.0.0.dylib \
  ${BUNDLE}/lib/libgimpmath-3.0.0.dylib "$PLUGIN"
install_name_tool -change @rpath/lib/libgimpmodule-3.0.0.dylib \
  ${BUNDLE}/lib/libgimpmodule-3.0.0.dylib "$PLUGIN"
install_name_tool -change @rpath/lib/libgimpthumb-3.0.0.dylib \
  ${BUNDLE}/lib/libgimpthumb-3.0.0.dylib "$PLUGIN"

  ${BUNDLE}/lib/libgimpui-3.0.0.dylib "$PLUGIN"

# Fix @rpath/lib/ entries for Homebrew libraries
install_name_tool -change @rpath/lib/libpng16.16.dylib \
  ${BREW}/opt/libpng/lib/libpng16.16.dylib "$PLUGIN"
install_name_tool -change @rpath/lib/libjpeg.8.dylib \
  ${BREW}/opt/jpeg-turbo/lib/libjpeg.8.dylib "$PLUGIN"
```

## Step 9 â€” Install the plugin

```bash
PLUGIN_SRC=~/PhotoApps/gmic-qt/gmic_gimp_qt.app/Contents/MacOS/gmic_gimp_qt
PLUGIN_DST="$HOME/Library/Application Support/GIMP/3.2/plug-ins/gmic_gimp_qt/gmic_gimp_qt"

mkdir -p "$HOME/Library/Application Support/GIMP/3.2/plug-ins/gmic_gimp_qt"
cp "$PLUGIN_SRC" "$PLUGIN_DST"
chmod +x "$PLUGIN_DST"
xattr -dr com.apple.quarantine "$PLUGIN_DST"
codesign --force --sign - "$PLUGIN_DST"
```

---

## Step 10 â€” Verify and launch

Test the plugin standalone (it should print a message saying it must be run by GIMP):

```bash
"$HOME/Library/Application Support/GIMP/3.2/plug-ins/gmic_gimp_qt/gmic_gimp_qt"
# Expected: "... is a GIMP plug-in and must be run by GIMP to be used"
```

Launch GIMP â€” G'MIC should now appear under the **Filters** menu.

## Troubleshooting

**Plugin doesn't appear in Filters after install:**
- Confirm the plugin directory is `~/Library/Application Support/GIMP/3.2/plug-ins/`
  (note `3.2`, not `3.0`) â€” verify in GIMP under Edit > Preferences > Folders > Plug-ins.
- Launch GIMP from Terminal to see error messages:
  `/Applications/GIMP.app/Contents/MacOS/gimp 2>&1 | grep -i "gmic\|error"`

**`dyld: Library not loaded` errors:**
- Run the plugin directly from Terminal to see which library is missing.
- Use `install_name_tool -change` to fix the path as shown in Step 8.
- Check all non-system library paths with:
  `otool -L "$HOME/Library/Application Support/GIMP/3.2/plug-ins/gmic_gimp_qt/gmic_gimp_qt"`

**`Symbol not found: _g_once_init_enter_pointer`:**
- The `-Wl,-alias` flags in Step 7 handle the GLib symbol mismatch between
  Homebrew headers and the bundle's older GLib. Ensure those flags are present
  in the qmake command when rebuilding.

**`main executable failed strict validation`:**
- This happens if the binary is modified after signing. Always re-sign with
  `codesign --force --sign -` after any `install_name_tool` changes.
- Do not attempt to patch the binary directly with a hex editor or Python â€”
  macOS Sequoia will reject the result even after re-signing.

**Rebuilding after a gmic-qt update:**
- Re-apply the patch in Step 6 if `host_gimp.cpp` has changed.
- Re-run Steps 7-9.
- The `gimp-3.0.pc` file and `gimpversion.h` do not need to be recreated.
